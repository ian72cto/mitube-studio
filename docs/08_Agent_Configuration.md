# MLC 개발 에이전트(MAO) 구성 및 지침

이 문서는 Antigravity 내에서 MLC 프로젝트를 개발할 에이전트들의 역할과 주요 구현 지침을 정의합니다.

## 1. 에이전트 역할 분담

### Agent Hardware (하드웨어/드라이버)
- **역할:** Camera2 API, JNI(C++), JVD(Peltier), HID(Gimbal) 제어.
- **핵심 목표:** 발열 관리 알고리즘 구현 및 4핀 렌즈 인식 인터페이스 최적화.

### Agent Streamer (송출/네트워크)
- **역할:** SRT/NDI|HX3 엔진 포팅, 듀얼심 본딩 알고리즘, 네트워크 페일오버 로직.
- **핵심 목표:** 4K 60fps 무손실 저지연 송출 안정성 확보.

### Agent Sync (동기화/AI)
- **역할:** AI STT 연동(감독 음성 분석), 시각 큐 증강현실 표시, 오디오 싱크 보정 엔진.
- **핵심 목표:** 감독의 지시 사항을 촬영자 화면에 지연 없이 시각화.

### Agent UI/UX (인터페이스/대시보드)
- **역할:** 안드로이드 3중 오버레이 UI, 웹 기반 CCU 대시보드(React).
- **핵심 목표:** 전문 방송 엔지니어가 한눈에 다수의 카메라를 제어할 수 있는 직관적 UI 제공.

## 2. 공통 개발 규칙
- **Performance First:** 영상 처리는 반드시 C++ 네이티브 레벨(NDK)에서 수행합니다.
- **Fail-Safe:** 모든 네트워크 통신은 연결 끊김 시 자동 재연결(Retry) 로직을 포함합니다.
- **Documentation:** 모든 함수와 모듈은 마크다운 주석을 포함하여 에이전트 간 지식 공유가 원활하게 합니다.

## 3. 초기 구현 우선순위
1. **Camera2 API Triple-Path:** 원본 영상을 3개로 분기하는 기초 파이프라인.
2. **WebSocket CCU Bridge:** 서버와 앱 간의 실시간 명령 주고받기.
3. **NDI/SRT Basic Stream:** 단일 결로 고화질 송출 테스트.
